# Pattern: Answer Ready Content

## Applicable Situation

Use when existing product facts, positioning, value proposition, proof, or category expertise must be turned into content that search engines and answer engines can parse, trust, and cite.

## Inputs

- ICP or audience
- Positioning and value proposition
- Product, company, or topic facts
- Proof assets and third-party evidence
- Target queries and questions
- Existing page, article, FAQ, schema, or documentation inventory

## Default Workflow

1. Start from approved positioning and value proposition; do not rewrite the GTM strategy.
2. Map target questions to pages, sections, FAQs, and proof assets.
3. Make entity facts explicit: names, categories, use cases, alternatives, integrations, pricing logic, limitations, and source dates.
4. Add answer-friendly structure: direct answers, comparison tables, FAQ, definitions, examples, and evidence blocks.
5. Add or specify structured data when appropriate: Article, FAQ, Product, Organization, Breadcrumb, Review, Dataset, or related schema.
6. Connect claims to proof: customer examples, benchmark data, documentation, third-party references, analyst mentions, or original research.
7. Define a refresh cadence for pages whose facts, screenshots, pricing, benchmarks, or recommendations decay.

## Branch Rules

- For launch support, use `launch-motion-design` for the launch plan and produce only the visibility checklist and content requirements.
- For outbound or sales messaging, use `value-prop-messaging` first, then structure the approved message for pages and answer surfaces.
- For pricing or packaging pages, flag machine-readability gaps but do not decide package boundaries.
- For agent-facing pages, evaluate discoverability and readability only; agent tool design belongs to Agent and Product skills.

## Anti-Patterns

- Creating "AI-friendly messaging" by inventing new positioning.
- Publishing generic FAQ sections with no source evidence.
- Adding schema that contradicts page content.
- Treating content freshness as optional for fast-changing AI or market claims.

## Acceptance Criteria

- The content answers target questions directly and cites proof.
- Entity facts are consistent across page copy, schema, FAQ, and source references.
- The brief separates GTM/Product decisions from visibility execution.
- Refresh requirements are explicit.

## Conflict Notes

AEO sources often push more content, while SEO documentation emphasizes structured, accurate, helpful pages. Prefer fewer pages with clear facts, proof, and schema over broad shallow coverage.
